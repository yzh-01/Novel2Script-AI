# 3 天开发路线图

---

## 评分权重复盘

| 考核点 | 权重 | 对应交付 |
|--------|:---:|------|
| A: AI 转换能力 | 40% | Prompt 工程 + 多类型适配 |
| C+D: Schema 文档 + 设计原因 | 30% | docs/YAML-Schema.md + docs/设计理念.md |
| B: 可编辑性 | 20% | 编辑器 + 校验 + 预览 + 溯源 |
| 整体完成度 | 10% | 闭环体验 + 零中断 |

**优先级公式：** 转换质量 ≈ Schema 文档 > 可编辑体验 > 功能数量

---

## Day 1 — 核心链路跑通

| 时段 | 任务 | 产出 | 验收 |
|------|------|------|------|
| **上午** | 项目骨架搭建 | `npm create next-app` + 目录结构 + 组件 stub | `npm run dev` 不报错 |
| **上午** | 输入页 UI | ChapterInput + ChapterList + genre/format 选择器 | 可增删章节，≥3 校验生效 |
| **下午** | ★ Prompt 工程 | 手写 System Prompt，明确转换规则 + 类型策略 | 用三体片段测试输出质量 |
| **下午** | Prompt 迭代 | 测试 3 种类型片段各 2 轮，修复幻觉和不一致 | 角色识别率 >80%，场景划分合理 |
| **晚上** | API 接口 | `/api/convert` + Claude SDK 接入 + Zod 校验 | 粘贴→点转换→控制台看到 YAML |

### Day 1 验收标准
粘贴 3 章《三体》片段 → `curl POST /api/convert` → 返回合法 YAML 剧本（至少 3 个角色 + 5 个场景）

---

## Day 2 — 编辑器 + Schema 文档

| 时段 | 任务 | 产出 | 验收 |
|------|------|------|------|
| **上午** | CodeMirror 6 集成 | YAML 语法高亮 + 行号 + 折叠 + 编辑同步 | 编辑器可编辑，改动反映到 state |
| **上午** | 前端 zod 校验 | `useEditorValidation` + 300ms 防抖 + 错误标红 | 删一个字符→300ms 后 lint 标红 |
| **下午** | ★ Schema 文档写作 | `YAML-Schema.md`（字段定义+设计原因+示例）| 独立可读，2000-3000 字 |
| **下午** | 设计理念文档 | `设计理念.md`（8 个为什么） | 独立可读，1500-2000 字 |
| **晚上** | 下载 + 预览 | DownloadButton + ScreenplayPreview（YAML→可读渲染） | 下载 `.yaml` → 预览面板显示角色名：对白 |
| **晚上** | 章节映射表 | SceneChapterMap（source_chapter→场景） | 展示"第1章→SC-001~004" |

### Day 2 验收标准
编辑器修改 YAML → 错误行标红 → 下载文件 → Schema 文档页可独立阅读

---

## Day 3 — 兜底 + 体验 + 测试

| 时段 | 任务 | 产出 | 验收 |
|------|------|------|------|
| **上午** | 两阶段返回 | `/api/convert` NDJSON 流 + ConversionProgress UI | 先看到角色列表（3s），再看到完整 YAML（20s） |
| **上午** | 错误处理 | Claude 超时/格式错→重试 1 次→友好报错 | 故意断网→不白屏→显示"重试"按钮 |
| **下午** | ★ 多类型测试 | 科幻（fixtures/sci-fi-sample.txt）+ 言情 + 历史 | 三种类型均输出合格剧本 |
| **下午** | Bug 修复 | 测试中发现问题→修→再测 | 无阻断性 bug |
| **晚上** | 终测 + README | 端到端走查 + README.md 定稿 + 部署准备 | 全流程无中断 |

### Day 3 验收标准
三种类型小说 → 稳定出剧本 → 编辑器可改 → 下载成功 → 文档齐备 → README 清晰

## 砍掉清单（不会做的）

| 功能 | 砍掉原因 |
|------|------|
| 暗色模式 | 3 小时投入，零评分回报 |
| 长文本分章调用 (>50K chars) | 风险高（ID 冲突/编号断裂），加字数上限提示代替 |
| 角色关系图 | 与四项考核点均无直接关联 |
| 多格式导出（Final Draft / PDF） | 口头说明架构可扩展即可 |
| 差异对比（AI 生成 vs 手动编辑） | 3 天窗口内做不好 |
| 用户系统 / 数据库 | MVP 范围外 |
| 流式输出 (SSE) | 两阶段方案已覆盖体验需求，成本 1/3 |

## 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|:---:|------|------|
| Claude API 输出格式不稳 | 中 | 高 | Zod 校验 + 重试 1 次 + 错误修复提示 |
| 某类型小说输出质量差 | 中 | 高 | Day 1 下午充分 Prompt 调优 + Day 3 下午留修复时间 |
| CodeMirror 6 集成耗时超预期 | 低 | 中 | Fallback：直接 `<textarea>` + 纯文本编辑（功能降级但不阻断） |
| 部署超时 | 低 | 低 | Vercel 免费版 10s → hobby 版 60s |
